# 1st
- 問題: [49. Group Anagrams](https://leetcode.com/problems/group-anagrams/)
- コメント集: [49. Group Anagrams](https://docs.google.com/document/d/11HV35ADPo9QxJOpJQ24FcZvtvioli770WWdZZDaLOfg/mobilebasic#h.e5dwa7yj3tv0)
- 条件
- 方針
  - strs を回して与えられた文字列をソートし、HashMap を検索する
    - ソートアルゴリズムとして、Collections vs Stream があるのでアルゴリズムを見に行く
  - 存在しなければソート後の文字列を key, ソート前の文字列を value として hashmap に保存する
  - 存在すれば、value に追加する
  - strs を走査し切ったら、hashmap にはそれぞれのアナグラムごとに key ,value が存在しているので、value を List に格納し直して返却すれば良い
- 時間計算量: strs.length = n, strs[i] = k とすると、O(n * k log k). 最悪計算量は定数を落として 10^6なので1sで収まる
- 空間計算量: O(N * k)、時間計算量と同様最悪 10^6
- HashMap の構造について
  - `containsKey()`: hash(引数)となるため、key が一意に定まるのでO(1)
  - `put()`: hash(key) でインデックスを計算、実際の値は LinkedList の構造で管理される、末尾に追加すれば良いので O(1)
- そのほか気になること
  - `toCharArray()`の挙動: char[] を新しくメモリ確保し、String の内部 char[] に新しい配列をコピーする。要するに String のイミュータビリティを守るため　
  - `Arrays.sort()`の挙動: Dual-Pivot QuickSort. pivot を2つ指定して、3領域に分けてソートする。計算量は O(n log n)

## 2nd 
- String のソートは String を一旦 char に unboxing して、ソートして、その後さらに String に boxing する必要がある
```java
class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        Map<String, List<String>> anagrams = new HashMap<>();
        
        for (String str : strs) {
            char[] c = str.toCharArray();
            Arrays.sort(c);
            String sorted = new String(c);
            if (anagrams.containsKey(sorted)) {
                anagrams.get(sorted).add(str);
            } else {
                List<String> temp = new ArrayList<>();
                temp.add(str);
                anagrams.put(sorted, temp);
            }
        }

        return new ArrayList<>(anagrams.values());
    }
}
```
